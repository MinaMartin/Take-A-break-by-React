Movies

All the paths starts with /movies --> Ex. for end point /watched the path becomes /movies/watched

Authentication token -which is valid for only 1 hour- must be included in the header of each request of the movies

End Points Request Method message Notes
/ GET Matched movies or All the movies Mentioning query parameter page = (Number) in the get request will trigger the pagination
but all movies are returned by default (/movies/ only without query parameter page) without pagination
query params name/releaseYear/genre are for search for Ex.
movies?releaseYear=2010 --> search by release Year
movies?genre=Horror --> search by genre
pagination Always on except when you request all the movies by movies/
/:movieId GET The movie You requested ---
/edit-movie/:movieId PATCH Movie Updated Successfully ---
/:movieId DELETE Movie Deleted Successfully ---
/watched GET Watched movies /movies/toBeWatched
/movies/toBeWatched?page=1 --> same result as the above end point
/movies/toBeWatched?page=2 --> page = 2
/toBeWatched GET ToBeWatched movies Same as watched
/add-movie POST Movie Created Successfully req body Ex.:
{"name":"Frozen","genre" : "Animation","releaseYear":2010}
movies are unwatched by default and year of watch by default is 0
if req body contain year of watch or watched = true that will update the default ofcourse
/addToWatchedList/:movieId PATCH Movie added to watched Movies movies/addToWatchedList/5e5beb0d69f38813b4ea1a78
req body Ex.: {"yearYouWatchedIn" : number}
if no request body yearYouWatchedIn will be the current year

Authentication

All the pathes starts with /auth --> Ex. for end point /login the path becomes /auth/login

End Points Request Method message Notes
/signUp POST User Signed up Successfully req body Ex.:
{"name":"...","email":"test@test.com","password":'123456',"confirmedPassword":'123456'}
the email field must contain a valid email address
If the same email found in the DB an error message occured ('Email Already Exists')
password with min length = 5 and max length = 12
name field cannot be empty
ofcourse password and confirmed password must be equal
/login POST User Logged in Successfully {"email":"test@test.com","password":'123456'}
If the email is not found in the DB A error message occured ('No User with this email')
password with min length = 5 and max length = 12
